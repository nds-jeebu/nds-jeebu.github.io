Markov Decision Processes

In my previous post, I off-handedly proposed several definitions and went right into describing an agent that wanted to maximize rewards. Before moving on, I want to quickly discuss a few key details of some of those definitions.

Sequential Decisions

When I introduced the trajectory: s0,a0,r1,s1,a1,r2,....,st−1,at−1,sts_0,a_0,r_1,s_1,a_1,r_2,....,s_{t-1},a_{t-1},s_ts0​,a0​,r1​,s1​,a1​,r2​,....,st−1​,at−1​,st​, I never said anything about decisions. I’m guessing it was kind of obvious that the agent starts in some state s0s_0s0​, takes some action a0a_0a0​, and ends up in the state s1s_1s1​. It repeats this process many times and you wind up with a trajectory.

Notice that this is a sequential decision process and there is an implicit assumption built into this process. Namely, that the agent can only take actions in the set of possible actions AAA. Furthermore, it may make only one action aaa that has an immediate reward rrr. After the action, the agent is left in the next state.

State Transition Probability

Recall the probability of ending up in state st+1s_{t+1}st+1​ after taking action ata_tat​ while in state sts_tst​: P(st+1∣st,at)P(s_{t+1}|s_t,a_t)P(st+1​∣st​,at​). I never called it this before, but this distribution represents the dynamics of the environment.

There are two implicit assumptions built into the environment here.

  1. When an agent takes an action, there is a probability it ends up in the state st+1s_{t+1}st+1​.
  2. The probability of ending up in state st+1s_{t+1}st+1​ depends only on the current state and current action.

Markov Decision Process

Our agent can make only sequential decisions with this property that the probability of ending up in the next state only depends on the current state. This is exactly what defines a Markov Decision Process (MDP) .

The Quality Function

When reading through the literature you’ll often see the Quality Function thrown around all over the place. I claim that we’ve already seen it, but it was hiding in plain sight.

Look at this equation again:
Vπ(s)=∑s′∈S[P(s′∣s,πA(a∣s))(R(s,πA(a∣s),s′)+Vπ(s′))]V^{\pi}(s)= \sum_{s' \in S}[P(s'|s, \pi_A(a|s))(R(s,\pi_A(a|s),s')+V^{\pi}(s'))]Vπ(s)=s′∈S∑​[P(s′∣s,πA​(a∣s))(R(s,πA​(a∣s),s′)+Vπ(s′))]
If instead of applying the policy πA(a∣s)\pi_A(a|s)πA​(a∣s), we leave this as a function of sss and aaa, we come to the Quality Function:

Define (Quality Function):
Qπ(s,a)=∑s′∈S[P(s′∣s,a)(R(s,a,s′)+Vπ(s′))]Q^{\pi}(s,a)= \sum_{s' \in S}[P(s'|s, a)(R(s,a,s')+V^{\pi}(s'))]Qπ(s,a)=s′∈S∑​[P(s′∣s,a)(R(s,a,s′)+Vπ(s′))]

The reason this is called the Quality Function is because ‘Value Function’ was already taken. It really is talking about the total expected future rewards.

Question: What does the Quality Function represent?
Answer: Check out the relationship below. Essentially, the Quality Function is the total expected future rewards given that the agent is in state sss and performs action aaa. After taking action aaa, the agent simply acts under policy π\piπ.

Observation (Relating the Value and the Quality):
Qπ(s,πA(a∣s))=Vπ(s)Q^{\pi}(s,\pi_A(a|s))=V^{\pi}(s)Qπ(s,πA​(a∣s))=Vπ(s)

While we’re here, I would like to establish one last definition in the vain of optimality.
Define (Optimal Quality Function):
Q∗(s,a)=∑s′∈S[P(s′∣s,a)(R(s,a,s′)+V∗(s′))]Q^{*}(s,a)= \sum_{s' \in S}[P(s'|s, a)(R(s,a,s')+V^{*}(s'))]Q∗(s,a)=s′∈S∑​[P(s′∣s,a)(R(s,a,s′)+V∗(s′))]

Extracting Policies:

Let’s assume that we are working through an example and I hand you an optimal Q∗(s,a)Q^{*}(s,a)Q∗(s,a). The standard procedure would be to use this to extract the policy π∗\pi^*π∗ .

Here’s how:
Procedure (Extract Optimal Policy from Optimal Quality Function):
πA∗(a∣s)=argmaxaQ∗(s,a)\pi_A^*(a|s)=argmax_{a}Q^{*}(s,a)πA∗​(a∣s)=argmaxa​Q∗(s,a)

We can also extract the optimal Value Function V∗(s)V^*(s)V∗(s) .

Procedure (Extract Value Function from Optimal Quality Function):
V∗(s)=maxaQ∗(s,a)V^*(s)=max_{a}Q^{*}(s,a)V∗(s)=maxa​Q∗(s,a)

It should be pretty clear that once we have the optimal quality function, we really have everything we need to have an optimal agent!
